CMS

CMS står for Content Management System. Det er applikasjon hvor flere brukere kan komme og gjøre ting som redigere, legge ut det man har gjort, som blir lagret i database og er tilgjenlig til deg og kollega ditt når som helst.

Det er en applikasjon som kan hjelpe folk til å lage nettside uten å ha noe kunnskap om koding, man kan tenke det som ferdig-laget nettsider. Det er front-end og back-end CMS. Front-end vil si ting som design, layout, farger osv. og Back-end er funksjonalitet som nettside-redigering og templates osv.

Når man skal bruke CMS så er det to valg, tradisjonelt CMS og headless CMS. Tradisjonelt CMS vil si at den har både front-end og back-end, altså at den har allerede det visuelle klart i tillegg til funksjonalitet. Headless CMS innholder bare back-end, altså funksjonalitet, men den har ikke noe med å gjøre for hvordan nettsiden skal se ut. Headless CMS bruke API til å koble til front-end hvor den bestemmer hva slags innhold i Headless CMS som skal vises. Den kan også bli brukt flere steder.

CCMS eller Component Content Management System er en CMS type som går an å bruke flere ganger. Den lagre innholdene i en form av komponent / modular for å lage en dokument. Den kan være video, bilde, avsnitt osv.

DMS eller Document Management System er en CMS type som lagre dokumenter. Den kan også brukes for å dele, opplaste sakt dokumentene uten å bruke fysisk ting som papir.

ECM eller Enterprise Content Management System er en CMS type som lagre, publiser og dele informasjon internt til eksakt person du vil sende til. Disse informasjonene blir lagres og kan bli søket opp etterhvert.

WCMS eller Web Content Management System er en CMS type som la brukere kontrollere innholdene i nettside uten å ha noe kunnskap om koding. Det gi forfatteren power til å redigere og administere nettsiden som han vil.

DAM eller Digital Asset Management System er en CMS type som lagre assets som video filer, dokumenter osv. I en cloud server og brukere få tilgang til.
